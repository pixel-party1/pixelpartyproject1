# Pixel Party Project

## Overview

Pixel Party is an iOS application built using Swift in Xcode.
This repository contains the full source code, project configuration, and test files for the application.

# Setting Up the Project

## Step 1 – Clone the Repository

Do **not download individual files**. Instead clone the entire repository.

In Xcode:

```
File → Clone Git Repository
```

Paste the repository URL:
PLEASE CHANGE USERNAME WITH YOUR OWN GITHUB USERNAME

```
https://github.com/USERNAME/pixelpartyproject1.git
```

Choose a location on your computer and clone the project.

---

## Step 2 – Open the Project

Once downloaded, open:

```
pixel party.xcodeproj
```

This will launch the full project in Xcode.

---

## Step 3 – Connect GitHub to Xcode

Each developer should link their GitHub account to Xcode.

Steps:

```
Xcode → Settings → Source Control -> Accounts
```

1. Click the **+** button
2. Select **GitHub**
3. Sign in with your GitHub account
4. THE TOCKEN MUST HAVE REPO, USER AND ADMIN:PUBLIC_KEY TICKED!!

---

# Working on the Project

When developing, follow this workflow.

### 1 Pull the latest changes

Before starting work:

```
Integrate -> Source Control → Pull
```

---

### 2 Make changes

Edit or add files within the **pixel party** folder.

---

### 3 Commit changes

After finishing work:

```
Integrate -> Source Control → Commit
```

Write a clear commit message.

Example:

```
Added player movement
Updated UI layout
Fixed collision detection
```

---

### 4 Push changes

Upload your commits to GitHub.

```
Integrate -> Source Control → Push
```

---

# Development Guidelines

* Always **pull before starting work**
* Commit changes frequently
* Write clear commit messages
* Avoid editing files directly on GitHub
* Perform development within Xcode
